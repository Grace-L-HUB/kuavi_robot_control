# 语音交互模块工作报告

## 1. 概述

本报告说明在「基于华为昇腾平台的轮式机器人语音交互与抓取递送系统」课程设计中，**语音交互子模块**（成员分工：语音识别 ASR + 语义理解 NLU）目前已完成的软件工作。实现策略为：**开发阶段在 PC 上使用 OpenAI Whisper 完成离线语音识别**，语义解析采用**可扩展的规则引擎**；**昇腾板端推理**通过统一的 ASR 后端接口预留，便于后续将 `whisper.om` 接入同一业务链路而无须重写上层代码。

---

## 2. 目标与范围

| 目标 | 说明 |
|------|------|
| 语音转文字 | 支持 wav/mp3 等常见格式（依赖 ffmpeg），默认中文 Whisper |
| 指令结构化 | 将自然语言转为下游可用的 JSON（动作、目标物体、颜色属性等） |
| 可替换后端 | ASR 可在「本地 Whisper」与「昇腾 OM（占位）」之间切换配置 |
| 端到端演示 | 支持「麦克风录制 → 识别 → JSON」完整流程 |

本阶段**未实现**：昇腾 CANN 上对 `whisper.om` 的实际推理代码（仅占位类）；视觉检测、机械臂控制由其他组员模块负责。

---

## 3. 系统架构与数据流

```
麦克风 / wav 文件
        │
        ▼
┌───────────────────┐
│ audio_record.py    │  （可选）16 kHz 单声道 WAV 录制
└─────────┬─────────┘
          ▼
┌───────────────────┐     config/asr.yaml
│ speech_recognition │ ──▶ 选择 ASR 后端：whisper_local / ascend_om(占位)
└─────────┬─────────┘
          ▼ 文本字符串
┌───────────────────┐
│ semantic_parser    │  规则匹配：颜色、物体、停止类指令
└─────────┬─────────┘
          ▼
   {"action", "target", "attribute", "location"}
```

对外统一入口包括：**仅文件**的 `speech_file_to_json`，以及 **麦克风录制** 的 `record_speech_to_json` / `record_then_parse`（后者同时返回识别文本，便于调试）。

## 4. 完成的工作内容

### 4.1 语音识别（ASR）

- **`vision_lim/asr_backends/base.py`**：抽象后端 `ASRBackend`；`AscendOmBackend` 占位，抛出 `NotImplementedError` 并提示后续在板端实现。
- **`vision_lim/asr_backends/whisper_local.py`**：调用 `openai-whisper`，支持在 `config/asr.yaml` 中配置模型规格（如 `tiny`）与语言（如 `zh`）。
- **`vision_lim/speech_recognition.py`**：读取配置，实例化对应后端，提供 `transcribe_file(audio_path)`。

### 4.2 语义理解（NLU）

- **`vision_lim/semantic_parser.py`**：规则版 `parse_instruction(text)`，将中文指令映射为结构化字典：
  - **action**：如 `fetch`（抓取递送）、`stop`（停止类关键词）、`unknown`；
  - **target**：物体英文类别码（如 `cup`、`ball`），与后续视觉类别表对齐；
  - **attribute**：颜色英文码（如 `red`、`blue`）；
  - **location**：预留字段，当前为 `null`。

### 4.3 语音链路整合

- **`vision_lim/voice_pipeline.py`**：
  - `normalize_asr_text`：规整 Whisper 输出空白；
  - `speech_file_to_json`：音频文件 → 任务 JSON；
  - `transcribe_then_parse`：返回 `(识别文本, 任务 JSON)`；
  - `record_then_parse` / `record_speech_to_json`：麦克风录制后与上述链路拼接。

### 4.4 麦克风录制（PC）

- **`vision_lim/audio_record.py`**：`record_wav_file`，使用 `sounddevice` 采集，输出标准 WAV（16 kHz、单声道、16 bit PCM），便于与板端格式约定一致。

### 4.5 脚本与配置

- **`config/asr.yaml`**：集中配置 ASR 后端与 Whisper 参数。
- **`requirements-asr.txt`**：`openai-whisper`、`PyYAML`、`numpy`、`sounddevice` 等语音链路依赖（与机器人侧依赖分离，按需安装）。
- **`scripts/demo_voice_pipeline.py`**：命令行演示（文本-only NLU、wav 文件、 `--record` 录制）。
- **`samples/README.md`**：测试音频目录说明与示例命令。

### 4.6 文档与仓库结构

- 根目录 **`readme.md`** 中「项目结构」已更新为实际目录名 **`vision_lim`**，并补充 `asr_backends`、`audio_record.py`、`voice_pipeline.py`、`config/asr.yaml`、`scripts`、`samples` 等说明。

## 5. 使用方法摘要

**依赖安装（语音开发环境）：**

```bash
pip install -r requirements-asr.txt
```

需在本机安装 **ffmpeg** 并加入 PATH（Whisper 依赖）。

**常用命令（在仓库根目录）：**

```bash
# 仅测语义（无需 Whisper）
python scripts/demo_voice_pipeline.py --text "递给我蓝色的球"

# 已有 wav：识别 + JSON
python scripts/demo_voice_pipeline.py samples/xxx.wav

# 麦克风录制后识别（默认约 5 秒）
python scripts/demo_voice_pipeline.py --record

# 录制并保存 wav
python scripts/demo_voice_pipeline.py --record -o samples/last.wav --seconds 6
```

**编程接口示例：**

```python
from vision_lim import speech_file_to_json, record_then_parse

task = speech_file_to_json("samples/demo.wav")
text, task = record_then_parse(duration_sec=5.0)
```

## 6. 测试情况说明

- 已在本地验证：**文本指令**可稳定输出预期 JSON。
- **端到端**：在安装 Whisper、ffmpeg、sounddevice 的前提下，`--record` 或 wav 文件路径可完成「语音 → 文本 → JSON」。
- 实际测试中出现：**Whisper 有时输出繁体中文**（例如「藍色的杯子」），而当前规则词典仅覆盖**简体**颜色词（如「蓝色」），可能导致 **`attribute` 为空**，但 **target** 仍可识别（如 `cup`）。属预期内的规则局限，非 ASR 完全失效。

## 7. 已知局限与后续工作

| 项目 | 说明 |
|------|------|
| 昇腾 ASR | 需在 `AscendOmBackend` 中接入 CANN 与 `ascend_models/whisper.om`，并在板子上将 `config/asr.yaml` 中 `backend` 改为 `ascend_om`。 |
| 繁简与标点 | 可在 NLU 前增加「繁体转简体」或扩充颜色同义词表；或对 Whisper 加强语言/后处理约束。 |
| 语义升级 | 可将 `parse_instruction` 内部替换为大模型 API（DeepSeek、通义等），保持输出 JSON  schema 不变即可对接下游。 |
| 唤醒词 | 项目文案中的「小助小助」等可在 `normalize_asr_text` 或 NLU 前做前缀裁剪。 |

---

## 8. 与其他模块的接口约定

- **输入**：ASR 输出为单一字符串；视觉/控制模块不应依赖具体 ASR 实现（Whisper / 昇腾）。
- **输出**：语义模块输出字典字段建议固定为 **`action`、`target`、`attribute`、`location`**，便于任务调度与 YOLO 类别、颜色筛选对齐。

## 9. 结论

目前已完成 **PC 端离线语音识别、规则语义解析、语音到 JSON 的流水线封装、麦克风录制与命令行演示**，并通过 **可替换 ASR 后端** 为华为昇腾板上的 `whisper.om` 推理预留扩展点。后续工作重点是在开发板上**实现并验证昇腾 ASR 后端**，并根据实测评测结果优化**文本规范化与 NLU 规则（或接入大模型）**，以提升复杂口语与繁简混写场景下的鲁棒性。
